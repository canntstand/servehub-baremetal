# servehub-baremetal
Проект представляет собой развёртывание реального веб-приложения на «голой» виртуальной машине с полным контролем каждого компонента на уровне ОС. Здесь нет контейнеров – только Ansible, systemd, пакеты и классическое администрирование Linux. Приложение `shared-chat` написано на Go и даёт живую страницу общего чата через WebSockets, а сообщения сохраняются в PostgreSQL.

## Цель проекта

Освоить фундамент системного администрирования: научиться настраивать продакшен-подобное окружение без контейнерной абстракции, понимать, как работают процессы, юниты, файрвол и резервное копирование, и добиться полной воспроизводимости среды с помощью Ansible.

## Как это выглядит

1. **Подготовка ВМ**: Чистый сервер (Ubuntu/Debian) – локально через Vagrant или на любом VPS.
2. **Код как конфигурация**: Весь сетап (пакеты, сервисы, конфиги, деплой) описан в Ansible-ролях.
3. **Автоматический деплой**: GitLab CI запускает `ansible-playbook site.yml` при пуше – роли проводят машину от свежей установки до полностью готового чата.
4. **Живое приложение**: Пользователи заходят на страницу, вводят ник (без регистрации, хранится только в текущей сессии) и тут же общаются в общей комнате через WebSocket. Сообщения пишутся в PostgreSQL и видны всем.
5. **Сопровождение**: Systemd следит за процессами, cron делает бэкапы, fail2ban защищает SSH, а ufw закрывает всё лишнее.

## Что используется

- **App**: Go-приложение `shared-chat` (WebSocket-чат, общая комната, ники на уровне сессии).
- **DB & Cache**: PostgreSQL (сообщения, лог чата), Redis (опциональный pub/sub для масштабирования WebSocket-нод).
- **Прокси**: Nginx (HTTPS через Let’s Encrypt, проксирование WebSocket-соединений).
- **Автоматизация и управление конфигурацией**: Ansible (роли, плейбуки), GitLab CI.
- **Безопасность**: ufw (deny all, разрешены только 80/443/22), fail2ban, доступ по SSH только по ключам.
- **Резервное копирование**: pg_dump + rsync по cron (ежедневные бэкапы с ротацией).
- **Инициализация и процессы**: systemd (юниты с лимитами ресурсов и автоперезапуском).

## Основной функционал

- **Полный веб-стек без Docker**: PostgreSQL, Redis, Nginx и сам Go‑сервис работают как нативные процессы ОС.
- **Живой чат с WebSockets**: Открытая общая страница, минималистичная идентификация по нику (без паролей, ник исчезает при очистки кеша браузера, сообщения остаются в БД).
- **Управление через systemd**: Каждый компонент – юнит с прописанными CPU/RAM-лимитами и политикой рестарта.
- **Сетевая безопасность на уровне хоста**: ufw закрыт по умолчанию, открыты только нужные порты; fail2ban блокирует попытки подбора SSH.
- **Автоматические резервные копии**: Ежедневные дампы PostgreSQL и копии конфигураций синхронизируются на удалённую машину или остаются локально с ротацией.
- **Идемпотентное развёртывание**: Один запуск `ansible-playbook` на свежей ВМ приводит её в целевое состояние; повторные запуски не вызывают побочных эффектов.
- **Интеграция с GitLab CI**: Пайплайн проверяет синтаксис Ansible и автоматически применяет изменения при пуше в ветку `main`.

## Примерная структура

```
servehub-baremetal/
├── ansible/                    # Ansible-конфигурация
│   ├── playbooks/
│   │   └── site.yml            # Основной плейбук
│   ├── roles/
│   │   ├── system/             # Базовые настройки ОС, ufw, fail2ban
│   │   ├── postgresql/         # Установка и настройка PostgreSQL
│   │   ├── redis/              # Установка Redis (опционально)
│   │   ├── nginx/              # Настройка обратного прокси и HTTPS
│   │   └── shared-chat/        # Сборка и запуск Go-приложения
│   ├── templates/              # Шаблоны конфигов и systemd-юнитов
│   └── inventory/              # dev/prod инвентори
│       ├── dev.yml
│       └── prod.yml
├── app/                        # Исходный код Go-приложения shared-chat
├── scripts/                    # Shell-скрипты (например, для cron-бэкапов)
├── config/                     # Готовые конфигурационные файлы
│   ├── nginx/
│   │   └── shared-chat.conf
│   └── cron/
│       └── backup.sh
└── .gitlab-ci.yml              # CI/CD пайплайн
```

## Ключевые фишки

- **Аппаратная прозрачность**: Вы видите каждый PID, каждый сокет, каждое правило iptables – никаких слоёв контейнеризации.
- **Навыки “голого” администрирования**: Уверенная работа с systemd, cron, ufw, правами, пакетами и отладка без Docker – то, что спрашивают на собеседованиях в DevOps и SRE.
- **Идентичное воспроизведение**: Ansible гарантирует, что любая новая ВМ станет точной копией описанного окружения (идемпотентность).
- **Простое, но живое приложение**: Чат с WebSocket даёт реальную нагрузку для мониторинга, а ники без регистрации не требуют сложной аутентификации, упрощая фокус на инфраструктуре.
- **Готовность к legacy и baremetal-средам**: Применимо к проектам, где контейнеризация запрещена или нецелесообразна, но автоматизация нужна не меньшая.
- **Дополняет DevOps-портфолио**: Вместе с servehub-IaC (Docker + Terraform) и servehub-monitor (наблюдаемость) даёт полное покрытие – от инфраструктуры как кода и мониторинга до классического сисадминства.
